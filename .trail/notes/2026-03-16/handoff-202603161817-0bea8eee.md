## Key Discovery: Two Conversation Flows Needed

The production tracer revealed that the Phase 2C plan's assumption (broker creates group, adds user by inbox ID) doesn't match how Convos works. Two flows are needed:

**Flow A — "Join my chat" (user invites broker):**
- User creates conversation in Convos, shares invite link (`popup.convos.org/v2?i=<base64url-protobuf>`)
- Broker parses invite tag, creates/selects identity, follows Convos join protocol (DM creator → wait for add → verify)
- Reference: `.reference/convos-cli/` and `.reference/convos-node-sdk/` join flow

**Flow B — "Join the agent's chat" (broker invites user):**
- Broker creates identity + group
- Broker generates a **Convos-compatible** invite URL (`popup.convos.org/v2?i=...`)
- User scans QR / opens link in Convos to join
- This is where the terminal QR code matters

Both flows require understanding the Convos invite tag format (base64url-encoded protobuf in `?i=`). Flow A is parsing; Flow B is generating.

Example invite URL from user: `https://popup.convos.org/v2?i=Cm8KPwET31HR_I4w...`
